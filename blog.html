Discuss the scope of var, let, and const

Var is globally scoped. That means I can access the var from any location of the .js file. Let’s take a complicated situation. Suppose i declare a variable with var in a function. I can acces the var outside the function as well. It’s call hoisting behaviour. This only happens to var variable.

On the other hand let and const cant be hoisting. That means, if i declare the let and const in a function scope it’s only be accessible within the function. The main difference between let and const are let can be re-declare where as const can’t.

Tell us the use cases of null and undefined

Undefined: In many cases i can get i defined. Such as: if i declare a variable but not assign a value then i’ll get undefined. On the other case, if i want to access an object property or array index that doesn’t exist i wll get undefined.

Null: I can intentionally set a variable value as null and then i’ll get null for the variable.

What do you mean by REST API?

REST API mostly used for CRUD (create, read, update, delete) operation. This operation can be done by using the following methods.

  1. POST: create operation
  2. GET: read operation
  3. PUT: update operation for existing value. If not available PUT method will create a new one.
  4. PATCH: update operation for existing value. Doesn’t create new one like PUT.
  5. DELETE: delete operation.
